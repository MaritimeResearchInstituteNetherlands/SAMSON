Route bound and non-route bound
-------------------------------

This **crossing** ships model is applied for the following conditions:

*  All assigned ships of a raster cell are denoted as unbounded and have their own link with begin and endpoint within the boundaries of this cell.
 
*  If the route bound link and the non-route bound link of the ship intersect, the number of encounters are computed with the route-bound crossing ships model.

*  The encounter angle must fulfil the following condition for the intersecting links: :math:`{\theta}_{lim}<{\theta}_{ij}<{\pi}-{\theta}_{lim}`.

*  The model is applied for each link and raster cell intersection in an area.

The number of encounters in the set time unit for a single traffic link in a single raster cell is computed as follows:

.. math::
    :label: eq_rn_num

    N_{rn}=\sum_{i=1}^{n}\sum_{j=1}^{m}P_{ij}\frac{Q_i}{u_iu_j{\Delta}t}u_{ij}

The :math:`{\Delta}t` in :math:numref:`eq_rn_num` corresponds with the :math:`{\Delta}t` (duration of the observation) in the :ref:`theory_traffic` section.

the relationship between the shortened link length and the maximum possible length given the link direction. The number of passages (:math:`Q_i`) are 
identical for the shortened and original link.

    :alt: ship_ship_bounded_unbounded
    :align: center

Index :math:`i` represents the ship types and sizes in de ship database. Counter :math:`j` represents all passed ships in the raster cell during the 
applicable time unit, which is set in :math:`Q_i` and :math:`{\Delta}t`.

Now the computation of :math:`P_{ij}` also depends on the encounter angle :math:`{\theta}_{ij}` and the crossing condition:
    
.. math::    
    :label: eq_rn_prop_cond
       
    P_{ij}   = \begin{cases}
                 \begin{split}
                                                 0 &\text{, for non intersecting links} \\
                   \frac{D_{ij}}{sin{\theta}_{ij}} &\text{, for intersecting links} 
                 \end{split}
               \end{cases}   

The encounter angle :math:`{\theta}_{ij}` is unique for each pair of intersecting links and so is the apparent collision diameter (:math:`D_{ij}`) and 
relative velocity (:math:`u_{ij}`). The computed encounters (:math:`N_{rn}`) must be assigned to the link and the raster cell with the ratios 
:math:`D_{i}/D_{ij}` and :math:`D_{j}/D_{ij}`. 

Non-route bound 
---------------

The **crossing** ships model is applied for the following conditions:

*  All assigned ships of a raster cell are denoted as unbounded and have their own link with begin and endpoint within the boundaries of this cell.
 
*  The number of encounters are computed with the route-bound crossing ships model for each unique pair of crossing links derived from the unbounded traffic .

*  The encounter angle must fulfil the following condition for the intersecting links: :math:`{\theta}_{lim}<{\theta}_{ij}<{\pi}-{\theta}_{lim}`.

*  The model is applied for each raster cell in an area.

The number of encounters in the set time unit for a single traffic link in a single raster cell is computed as follows:

.. math::
    :label: eq_nn_num
    
    N_{nn}=\sum_{i=1}^{m}\sum_{j=i+1}^{m}P_{ij}\frac{1}{u_iu_j{\Delta}t^2}u_{ij}

The calculation of :math:`N_{nn}` is performed for each raster cell in an area. The same conditions for :math:`P_{ij}` apply as for the :math:`N_{rn}` 
computation in the previous section.

.. math:: 
    :label: eq_nn_prop_cond
    
    P_{ij}   = \begin{cases}
                 \begin{split}
                                                 0 &\text{, for non intersecting links} \\
                   \frac{D_{ij}}{sin{\theta}_{ij}} &\text{, for intersecting links} 
                 \end{split}
               \end{cases}
