Drift Collision Quantification#

General#

Objective:

Verify that the the calculated drift frequency is consistent with the theoretical model

Criteria:

The calculated drift frequency should deviate less than 3% from the theoretical value

In this test case, the weather is defined to be bft 5 going to east. One link (south to north) and several square objects perpendicular to the link are defined. Current and tide are set to 0. In this special case, it is relatively easy to calculate the collision frequency due to drifting, so that C++ SAMSON can be compared with the theoretic ‘true’ values.
